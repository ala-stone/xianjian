# 多账号快速配置指南

## 📌 问题说明

**问题**: 之前脚本只处理第一个账号，即使配置了多个账号也只运行一个。

**已修复**: 脚本现在支持多账号批量签到，会依次处理所有账号。

---

## 🚀 快速开始

### 步骤 1: 获取账号信息

使用圈X抓包功能获取以下参数：

**必填参数:**
- `username` - 手机号或账号
- `password` - 密码
- `role_id` - 角色ID
- `role_name` - 角色名
- `server_id` - 服务器ID
- `server_name` - 服务器名

**通用参数（通常固定）:**
- `platform` - "android" 或 "ios"
- `page_id` - "8"
- `game_id` - "69"
- `app_id` - "58"

> 💡 详细获取方法请参考 `GET_ACCOUNT_INFO.md`

### 步骤 2: 配置账号

编辑 `xianjian_signin_final.js` 文件：

```javascript
const ACCOUNTS = [
    // 账号 1
    {
        username: "手机号1",
        password: "密码1",
        role_id: "角色ID1",
        role_name: "角色名1",
        server_id: "服务器ID1",
        server_name: "服务器名1",
        platform: "android",
        page_id: "8",
        game_id: "69",
        app_id: "58"
    },

    // 账号 2
    {
        username: "手机号2",
        password: "密码2",
        role_id: "角色ID2",
        role_name: "角色名2",
        server_id: "服务器ID2",
        server_name: "服务器名2",
        platform: "ios",
        page_id: "8",
        game_id: "69",
        app_id: "58"
    }

    // 可以继续添加更多账号...
];
```

### 步骤 3: 测试运行

1. 上传脚本到圈X
2. 手动运行一次
3. 查看日志，确认所有账号都正常

### 步骤 4: 设置定时任务

在圈X配置文件中添加：

```ini
[Task_local]
# 仙剑H5自动签到 - 每天8点
0 8 * * * https://你的脚本地址/xianjian_signin_final.js, tag=仙剑签到, enabled=true
```

## 🔧 常见问题

### Q1: 为什么配置了多个账号但日志只显示一个？

**A:** 确保你使用的是更新后的 `xianjian_signin_final.js`，旧版本只处理第一个账号。

### Q2: 某个账号失败会影响其他账号吗？

**A:** 不会。每个账号独立处理，一个失败不影响其他账号。

### Q3: 如何知道每个账号是否签到成功？

**A:** 查看日志输出，每个账号都有独立的结果显示。

### Q4: 可以添加多少个账号？

**A:** 没有限制，可以添加任意数量的账号。

### Q5: 不同账号可以来自不同服务器吗？

**A:** 可以。每个账号可以配置不同的服务器信息。
