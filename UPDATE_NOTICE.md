## 🔍 问题说明

**你遇到的情况**:
- 配置了 2 个账号
- 脚本只执行了 1 个账号
- 日志只显示一个账号的签到结果

**原因**:
之前的脚本代码只处理 `ACCOUNTS[0]`（第一个账号），即使你配置了多个账号，也只会对第一个账号进行签到。

**修复**:
已将脚本更新为遍历所有账号，现在可以正确处理多账号批量签到了！

## 🚀 如何更新

1. **下载新版本脚本**
   - 文件名: `xianjian_signin_final.js`
   - 确保下载的是 v1.1 版本

2. **上传到服务器**

   - 或使用 GitHub Raw 链接

3. **更新圈X配置**
   - 打开圈X配置文件
   - 找到仙剑签到任务
   - 更新脚本URL为新的地址

### 方法二：直接修改现有脚本

如果你不想下载新文件，可以按照以下步骤修改：

1. 打开 `xianjian_signin_final.js`
2. 找到 `main()` 函数（大约在第31行）
3. 将以下代码：

```javascript
const account = ACCOUNTS[0];
log(`【账号】${account.username}`);
```

替换为：

```javascript
// 遍历所有账号
for (let i = 0; i < ACCOUNTS.length; i++) {
    const account = ACCOUNTS[i];
    log(`【账号 ${i + 1}/${ACCOUNTS.length}】${account.username}`);
```

4. 将代码末尾的 `}` 括号之前添加账号分隔符：

```javascript
// 账号之间添加分隔符（最后一个账号除外）
if (i < ACCOUNTS.length - 1) {
    log("\n" + "-".repeat(40) + "\n");
}
}
```

**注意**: 建议直接使用新的脚本文件，手动修改可能出错。

## 📝 配置多账号

### 1. 获取账号信息

新版本需要完整的账号参数，不只是用户名和密码。

**必需参数**:
- `username` - 账号/手机号
- `password` - 密码
- `role_id` - 角色ID
- `role_name` - 角色名
- `server_id` - 服务器ID
- `server_name` - 服务器名
- `platform` - 平台（android/ios）
- `page_id` - 页面ID（通常为8）
- `game_id` - 游戏ID（通常为69）
- `app_id` - 应用ID（通常为58）

**如何获取参数**:
请参考 `GET_ACCOUNT_INFO.md` 文档，详细介绍了如何通过圈X抓包获取这些参数。

## ❓ 常见问题

### Q1: 我只有一个账号，需要更新吗？

**A:** 不需要强制更新，v1.0 版本仍然可以正常使用单账号签到。但建议更新到 v1.1，因为新版本有更好的日志输出和用户体验。

### Q2: 更新后需要重新配置账号吗？

**A:** 如果你的配置格式与新版本一致（包含所有必需参数），不需要重新配置。如果之前只有用户名和密码，需要补充其他参数。

### Q3: 如何获取 role_id 和 server_id？

**A:** 需要通过抓包获取。请参考 `GET_ACCOUNT_INFO.md` 文档，详细介绍了使用圈X抓包功能获取这些参数的方法。

### Q4: 更新后为什么日志显示"账号数量: 1"？

**A:** 说明你只配置了一个账号。这是正常的。如果需要添加更多账号，请按照配置示例添加。

### Q5: 某个账号签到失败会影响其他账号吗？

**A:** 不会。每个账号独立处理，一个失败不影响其他账号的签到。

### Q6: 我不知道如何抓包，怎么办？

**A:** 请仔细阅读 `GET_ACCOUNT_INFO.md` 文档，里面有详细的步骤说明。如果仍然不清楚，可以尝试使用游戏内的信息（角色名、服务器名），但 role_id 和 server_id 仍需通过抓包获取。
