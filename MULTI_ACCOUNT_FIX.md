# 多账号配置修复说明

## 问题描述

之前的脚本虽然定义了 `ACCOUNTS` 数组支持多账号配置，但实际上只处理了第一个账号：

```javascript
const account = ACCOUNTS[0];  // 只取第一个账号
```

这导致即使用户在配置中添加了多个账号，脚本也只会对第一个账号进行签到。

## 修复内容

### 1. 修改 `xianjian_signin_final.js`

已将单账号处理改为循环处理所有账号：

```javascript
// 遍历所有账号
for (let i = 0; i < ACCOUNTS.length; i++) {
    const account = ACCOUNTS[i];
    log(`【账号 ${i + 1}/${ACCOUNTS.length}】${account.username}`);
    
    // 执行登录、绑定角色、检查签到状态、签到等操作
    // ...
    
    // 账号之间添加分隔符
    if (i < ACCOUNTS.length - 1) {
        log("\n" + "-".repeat(40) + "\n");
    }
}
```

## 如何配置多个账号

### 方法一：直接修改脚本

1. 打开 `xianjian_signin_final.js`
2. 找到 `const ACCOUNTS = [...]` 配置部分
3. 按照现有格式添加更多账号：
   ```javascript
   const ACCOUNTS = [
       {
           username: "账号1",
           password: "密码1",
           role_id: "角色ID1",
           role_name: "角色名1",
           server_id: "服务器ID1",
           server_name: "服务器名1",
           platform: "android",
           page_id: "8",
           game_id: "69",
           app_id: "58"
       },
       {
           username: "账号2",
           password: "密码2",
           role_id: "角色ID2",
           role_name: "角色名2",
           server_id: "服务器ID2",
           server_name: "服务器名2",
           platform: "android",
           page_id: "8",
           game_id: "69",
           app_id: "58"
       }
       // 可以继续添加更多账号...
   ];
   ```

### 方法二：获取账号信息

如果不知道如何获取账号的完整信息，可以使用以下方法：

1. **在游戏中登录账号**
2. **使用抓包工具（如圈X的抓包功能）**
3. **查看以下请求的参数**：
   - 登录请求：`/snail/login/account`
   - 角色绑定请求：`/game/binds`
4. **从请求参数中提取以下信息**：
   - `role_id`: 角色ID
   - `role_name`: 角色名称
   - `server_id`: 服务器ID
   - `server_name`: 服务器名称

## 账号参数说明

| 参数 | 说明 | 示例 | 是否必需 |
|------|------|------|----------|
| `username` | 手机号或游戏账号 | "13569284920" | 是 |
| `password` | 登录密码 | "cgm19930716" | 是 |
| `role_id` | 角色ID（数字字符串） | "1410682267143104932" | 是 |
| `role_name` | 角色名称 | "node" | 是 |
| `server_id` | 服务器ID（数字字符串） | "20047" | 是 |
| `server_name` | 服务器名称 | "Q0047 身世浮沉" | 是 |
| `platform` | 平台类型 | "android" 或 "ios" | 是 |
| `page_id` | 页面ID（通常固定为8） | "8" | 是 |
| `game_id` | 游戏ID（通常固定为69） | "69" | 是 |
| `app_id` | 应用ID（通常固定为58） | "58" | 是 |

## 注意事项

1. **账号数量没有限制**，可以根据需要添加任意数量的账号
2. **账号之间会自动添加分隔符**，方便区分每个账号的执行结果
3. **统计信息会汇总所有账号的结果**（成功数、失败数、跳过数）
4. **通知消息也会包含汇总信息**
5. **如果某个账号执行失败**，不会影响其他账号的签到
6. **建议定期检查日志**，确保所有账号都正常签到
