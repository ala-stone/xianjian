# 仙剑H5自动签到脚本使用说明

## 脚本简介

这是一个专为 Quantumult X (圈X) 设计的仙剑H5自动签到脚本，支持账号密码登录和自动签到功能。

## 功能特点

- ✅ 支持账号密码登录
- ✅ 自动每日签到
- ✅ 支持多账号批量签到
- ✅ 支持光子服和正式服切换
- ✅ 签到结果通知
- ✅ 详细的日志输出

## 使用步骤

### 第一步：配置账号信息

打开 `xianjian_signin.js` 文件，修改账号配置区域：

```javascript
// 服务器选择: "photon" = 光子服, "formal" = 正式服
const SERVER_TYPE = "formal";

// 账号配置（支持多个账号）
const ACCOUNTS = [
    {
        username: "your_account_1",  // 替换为你的账号
        password: "your_password_1",  // 替换为你的密码
        server: "formal"              // formal=正式服, photon=光子服
    },
    // 添加第二个账号
    {
        username: "your_account_2",
        password: "your_password_2",
        server: "photon"
    }
];
```

**重要提示：**
- 请将 `username` 和 `password` 替换为你的实际账号和密码
- `server` 字段可选值为：`formal`（正式服）或 `photon`（光子服）
- 可以添加多个账号，脚本会自动依次执行签到

### 第二步：上传脚本到圈X

**方法一：使用本地文件**
1. 将 `xianjian_signin.js` 文件上传到你的服务器或云存储
2. 确保可以通过 HTTP/HTTPS 访问该文件
3. 获取脚本的完整 URL 地址

**方法二：使用 GitHub**
1. 将脚本上传到 GitHub 仓库
2. 使用 GitHub Raw 获取可访问的 URL
3. 例如：`https://raw.githubusercontent.com/你的用户名/仓库名/main/xianjian_signin.js`

### 第三步：配置圈X

1. **打开 Quantumult X**
2. **点击右下角的配置文件图标（小风车）**
3. **点击"配置文件编辑"**
4. **找到 `[Task_local]` 部分，添加以下配置：**

```ini
# 仙剑H5自动签到
0 8 * * * https://你的脚本地址/xianjian_signin.js, tag=仙剑签到, enabled=true
```

**配置说明：**
- `0 8 * * *`：表示每天早上 8:00 执行签到（可修改为任意时间）
  - 格式：`分 时 日 月 周`
  - 例如：`30 9 * * *` = 每天 9:30 执行
  - `*/30 * * * *` = 每30分钟执行一次
- `tag=仙剑签到`：在圈X任务列表中显示的名称
- `enabled=true`：启用该任务

5. **保存配置**
6. **点击"调试"->"构造请求"查看任务是否加载成功**

### 第四步：测试运行

1. 在圈X任务列表中找到"仙剑签到"任务
2. 手动点击运行一次
3. 查看日志输出和通知
4. 确认签到成功后，即可等待自动执行

## 高级配置

### 自定义签到时间

修改 cron 表达式来自定义执行时间：

| 表达式 | 说明 |
|--------|------|
| `0 8 * * *` | 每天 8:00 |
| `30 9 * * *` | 每天 9:30 |
| `0 8,12,18 * * *` | 每天 8:00, 12:00, 18:00 |
| `0 9 * * 1-5` | 工作日（周一至周五）9:00 |
| `0 0 1 * *` | 每月1号 0:00 |

## 常见问题

### Q1: 脚本无法运行？

**解决方法：**
1. 检查脚本 URL 是否可访问（建议使用 HTTPS）
2. 检查圈X是否已启用 Task 功能
3. 查看圈X日志，确认是否有错误信息
4. 确认账号密码是否正确

### Q2: 登录失败？

**可能原因：**
- 账号或密码错误
- 服务器类型选择错误（光子服/正式服）
- 账号被封禁或异常

**解决方法：**
1. 在网页版手动登录验证账号密码
2. 确认选择的 `server` 类型是否正确
3. 查看脚本日志中的错误信息

### Q3: 签到失败？

**可能原因：**
- 已经签到过（每天只能签到一次）
- 登录 token 过期
- 网络连接问题

**解决方法：**
1. 检查签到时间（建议设置在每日早上）
2. 查看日志中的详细错误信息
3. 检查网络连接

### Q4: 如何查看详细日志？

**方法：**
1. 在圈X中点击"调试"
2. 选择"日志"
3. 搜索"仙剑"关键字
4. 查看脚本输出的详细日志

## 接口说明

### 登录接口
- URL: `https://h5.qingcigame.com/api/login`
- 方法: POST
- 参数:
  ```json
  {
    "login_type": "password",
    "account": "账号",
    "password": "密码",
    "server": "formal"
  }
  ```
- 返回: 包含 token 的响应

### 签到接口
- URL: `https://h5.qingcigame.com/api/signin`
- 方法: POST
- Headers:
  - `Authorization: Bearer {token}`
- 参数:
  ```json
  {
    "server": "formal"
  }
  ```
- 返回: 签到结果和累计天数
